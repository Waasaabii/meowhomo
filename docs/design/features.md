# MeowHomo 功能详情设计

本篇详细规约了系统需要实现的核心功能及 5 大页面的展现逻辑。

## 1. 五大页面规约

### 1.1 仪表板 (Dashboard)
- **核心作用**：当前全局状态的第一总览。
- **展现内容**：流量统计图、连接数目、迷你版实时日志（终端风猫娘日志）、未读重要通知组件。

### 1.2 节点页 (Nodes)

**导入方式（3 种）**：
1. **订阅导入**：输入 Clash 订阅 URL，自动拉取并解析节点列表。
2. **手动导入**：粘贴单条 ss:// / vmess:// / trojan:// 链接。
3. **本机节点创建**：在服务器上直接创建代理入站（trojan 协议优先）。
   - 支持 **IP 直连模式**（纯 IP + 自签证书）和 **域名模式**（ACME 自动签发）。
   - 创建后自动生成：直连链接、QR 码、Clash 订阅 URL（含 token 鉴权）。

**统一列表**：摒弃多 Tab，将"订阅、手动、本机"按来源分组展现。顶部悬浮提供【+ 添加 ▼】下拉菜单。

**列表 + 地图联动**：
- **桌面端**：上 50% 列表 + 下 50% CesiumJS 地球，中间可拖拽分隔线调节比例。
- **全屏交互**：点击地图全屏后，地图 100% 铺满，列表变为左侧浮动面板。点击列表节点 → 地球 FlyTo 并高亮弧线。点击地球标记 → 列表自动滚动高亮对应节点。**双向联动**。
- **移动端**：隐藏 3D 地球，以国旗图标 + 纯列表替代，节省性能。

### 1.3 策略页 (Strategies)

**5 种策略类型**：

| 类型 | 说明 |
|------|------|
| `priority` | 按优先级顺序尝试节点，高优先级不可用时自动降级 |
| `speed-first` | 定期测速，自动选择延迟最低的节点 |
| `sticky-random` | 随机选择后粘性保持，断线才重新随机 |
| `manual` | 手动指定固定节点，不自动切换 |
| `relay` | 链式代理，流量依次经过多个节点（代理链） |

**通用策略能力**：
- **粘性路由 (Sticky)**：同一目标地址始终走同一节点，避免 IP 频繁变化。
- **优先级调整**：拖拽排序节点优先级。
- **健康检测**：定时 URL 测试（默认 http://www.gstatic.com/generate_204），不可达自动标记。
- **自动切换**：健康检测失败后自动切换到下一可用节点。
- **回切 (Fallback)**：原节点恢复后自动回切回高优先级节点。

**桌面端交互**：左右双栏 — 左侧策略列表（带状态/延迟/开关），右侧详情面板：
- 拖拽排序节点优先级
- `export http_proxy=xxx` 终端命令一键复制
- 对外暴露配置（独立端口 + 认证 + IP 白名单）

**移动端交互**：手风琴式卡片，展开显示节点列表和快捷操作（测速/切换/开关）。

### 1.4 内核页 (Engine)
- 展示 mihomo 实时状态（版本、运行时长、内存占用、连接数）。
- 提供重启引擎、重载配置、查看原始配置文件等操作。
- 数据全部通过 gRPC 实时获取。

### 1.5 设置页 (Settings)
包含以下 Tab 页：
1. **基础设置**：面板端口、语言、主题。
2. **域名与 TLS**：Let's Encrypt ACME 自动证书签发 + HTTPS 强制跳转。支持 IP 直连（自签证书）和域名模式。
3. **反向代理**：内置 Axum 网关转发，可与 Nginx 共存。
4. **安全保护**：JWT 登录防护 + 请求限流。
5. **系统备份**：一键导出/导入，支持 JSON / YAML / ZIP（含证书）三种格式。定时自动备份到 `backups/` 目录。
6. **系统信息**：mihomo 版本检查 + 上游更新提示。

## 2. 核心功能矩阵

| 功能域 | 详细说明 |
|--------|---------|
| **节点管理** | 3 种导入 + 本机创建（trojan 优先, IP/域名双模式）+ 直出链接/QR 码 |
| **策略引擎** | priority / speed-first / sticky-random / manual / relay 五种模式 |
| **通用策略** | 粘性路由 + 优先级拖拽 + 健康检测 + 自动切换 + 回切 |
| **对外暴露** | 每策略独立端口开关 + 认证 + IP 白名单 + 订阅链接（token 鉴权）|
| **安全认证** | 三层安全：① Dashboard JWT ② 代理 Authorization ③ 订阅 Token |
| **TLS 证书** | IP 直连（自签）/ 域名（ACME Let's Encrypt 自动签发 + 443 反代）|
| **运行时保护** | 端口冲突自动规避 + 无缝迁移 + SSE 推送 + 禁止全页刷新 |
| **日志系统** | 8 猫娘角色主题日志 + mihomo 原生日志拦截转化 + 过滤/搜索/暂停/下载 |
| **数据备份** | JSON / YAML / ZIP 全格式 + 定时自动备份 |
| **国际化** | react-i18next 框架，默认中文，保留多语言扩展能力 |
| **双端适配** | 桌面（shadcn + CesiumJS 3D 地球）+ 移动（Shadowrocket 风卡片列表）|

## 3. 端口冲突规避 (Auto Port Evading)

- 启动 Listener 时，若默认端口（如 `10001`）被占用，自动顺延检测空闲端口，启动后通过 SSE Toast 推送"已顺延至 10005 端口"。
- **运行时无缝迁移**：发现被挤占 → 先新建空闲 Listener → 成功后重定向策略绑定 → 最后撤下旧端口，保证热更不断线。

## 4. 三层安全认证 (Three-Tier Security)

1. **Dashboard 登录**：JWT Token 签发，支持过期刷新，保护 Web 管理面板。
2. **代理认证**：HTTP/Socks5 代理端口的 Authorization 基础鉴权，防止未授权使用。
3. **订阅 Token**：对外派发 Clash 订阅 URL 时的独立 Token 签发，每个订阅链接有独立 token，支持撤销。

## 5. 国际化 (i18n)

- 基于 `react-i18next`，代码保留多语言能力。
- 当前版本**默认且仅提供中文**内容。
